# RNF Retention Engine

The Retention Engine defines the behavioral systems that keep users returning to RNF daily.

RNF is not a passive habit tracker.

It is a **structured behavioral system** designed to reinforce consistency through identity, progression, and feedback.

---

# Core Principle

Retention comes from a repeating loop.

Loop structure:

Trigger  
→ Action  
→ Reward  
→ Progress  
→ Identity Reinforcement  
→ Repeat

RNF must reinforce this loop every day.

---

# The RNF Daily Loop

1. User receives a trigger
2. User opens the app
3. User completes habits
4. User receives rewards
5. Progress is updated
6. Identity reinforcement occurs

Loop repeats daily.

---

# Trigger Layer

Triggers remind users to return.

RNF uses two types of triggers.

## External Triggers

Morning notification

Example message:

"Start your day intentionally."

Evening reflection notification

Example message:

"Plan tomorrow before it arrives."

These triggers establish routine.

---

## Internal Triggers

Internal triggers form when users associate RNF with:

discipline  
self-improvement  
identity growth

Over time users open RNF automatically.

---

# Action Layer

The action must be simple.

Home screen displays:

Today's habits.

Completion should require:

< 2 taps.

Low friction increases compliance.

---

# Reward Layer

RNF provides multiple reward types.

These rewards reinforce behavior.

---

## Immediate Rewards

XP gain

Example:

+10 XP

Visual feedback:

small XP animation  
light haptic feedback

Immediate rewards reinforce habit completion.

---

## Progress Rewards

Level increases.

Example:

LEVEL 5 → LEVEL 6

Progress bars reinforce forward movement.

---

## Identity Rewards

Titles represent identity milestones.

Examples:

Ember Initiate  
Forged Disciple  
Iron Ascendant

Titles reinforce identity transformation.

---

## Streak Rewards

Streaks reward consistency.

Example:

7 Day Streak

Breaking streaks creates loss aversion.

Users want to maintain streaks.

---

# Forgiveness System

The forgiveness system prevents discouragement.

Users can earn forgiveness tokens.

Example:

1 forgiveness per 30 days.

Forgiveness protects streaks without removing accountability.

---

# Evolution Milestones

Evolution tiers represent major transformations.

Example:

Disciple → Awakened → Ascendant → Warlord → Apex

These events should feel significant.

They reinforce identity growth.

---

# Quest System

Quests introduce variation.

Instead of repeating identical habits, RNF generates tasks based on weak stats.

Example:

Low focus stat → reading quest.

This prevents boredom.

---

# Variable Rewards

RNF occasionally introduces surprise rewards.

Examples:

bonus XP  
special title unlock  
hidden achievement

Variable rewards increase engagement.

---

# Progress Visualization

Users must see progress clearly.

RNF provides:

XP bars  
level indicators  
radar stat charts  
calendar tracking

Visible progress increases motivation.

---

# Calendar Reinforcement

The calendar creates visual accountability.

GitHub-style progress grid.

Color meaning:

complete → green  
partial → light green  
missed → grey  
forgiven → green with dot

Users want to maintain visual consistency.

---

# Identity Formation

The most powerful retention factor is identity.

RNF encourages users to think:

"I am a disciplined person."

Identity reinforcement happens through:

titles  
evolutions  
skill trees  
streak milestones

---

# Habit Expansion

New habits unlock gradually.

Example:

Week 1 → hydration + diet

Week 2 → add walking

Week 3 → add journaling

Gradual complexity prevents overwhelm.

---

# Challenge Structure

The 90-day challenge provides long-term direction.

Users commit to:

90 days of discipline.

This increases psychological investment.

---

# Loss Aversion

People strongly avoid losing progress.

RNF uses:

streak tracking  
calendar visualization  
XP accumulation

These systems encourage users to maintain consistency.

---

# Social Expansion (Future)

Future systems may include:

guilds  
shared challenges  
leaderboards

Social pressure increases retention.

---

# Retention Metrics

Key metrics to monitor.

Day 1 retention

Percentage of users returning next day.

Target:

≥ 50%

---

Day 7 retention

Percentage of users returning after one week.

Target:

≥ 40%

---

Day 30 retention

Percentage of users still active after one month.

Target:

≥ 25%

---

# Behavioral Philosophy

RNF must never rely on:

guilt  
shame  
fear

Instead it reinforces:

discipline  
growth  
identity

Users should feel they are forging a stronger version of themselves.

---

# Design Rule

Every feature must support the retention loop.

If a feature does not strengthen the loop, it should not be added.
